# 音频同步：adelay 重计算方法

当视频片段时长变化时（如重新生成部分镜头），音频延迟必须重新计算。

## 公式

```
镜号N的adelay值 = sum(镜头1到N-1的帧数) / fps × 1000 (毫秒)

adelay值 = 累计帧数 / 16 × 1000
```

## 示例计算

12镜管线（16fps），有对白的镜头：05, 07, 08, 10, 11, 12

| 镜号 | 帧数 | 时长(s) | 累计开始时间(ms) |
|------|------|---------|-----------------|
| 01 | 81 | 5.0625 | 0 |
| 02 | 65 | 4.0625 | 5063 |
| 03 | 81 | 5.0625 | 9125 |
| 04 | 49 | 3.0625 | 14188 |
| **05** | 49 | 3.0625 | **17250** |
| 06 | 49 | 3.0625 | 20313 |
| **07** | 65 | 4.0625 | **23375** |
| **08** | 65 | 4.0625 | **27438** |
| 09 | 97 | 6.0625 | 31500 |
| **10** | 65 | 4.0625 | **37563** |
| **11** | 65 | 4.0625 | **41625** |
| **12** | 97 | 6.0625 | **45688** |

## ffmpeg 合并模板

```bash
ffmpeg -y \
  -i shot01.mp4 -i shot02.mp4 ... -i shot12.mp4 \
  -i shot05_voice.wav -i shot07_voice.wav ... \
  -i bgm.flac \
  -filter_complex "
    [0:v]...[11:v]concat=n=12:v=1:a=0[v];
    [12:a]adelay=17250|17250[a5];
    [13:a]adelay=23375|23375[a7];
    [14:a]adelay=27438|27438[a8];
    [15:a]adelay=37563|37563[a10];
    [16:a]adelay=41625|41625[a11];
    [17:a]adelay=45688|45688[a12];
    [a5][a7][a8][a10][a11][a12]amix=6:normalize=0[voices];
    [18:a]volume=0.3[bgm];
    [voices][bgm]amix=2:normalize=0[a]
  " -map "[v]" -map "[a]" -c:v libx264 -preset fast -crf 23 \
  -c:a aac -b:a 192k -ar 44100 -shortest output.mp4
```

## 常见错误

- **音频提前/延后**：用了旧的 adelay 值，没有按最新帧数重算
- **只改视频不改音频**：换了视频源但 adelay 保持不变 → 不同步
- **adelay 写死**：不要在脚本里硬编码 ms 值，应该动态计算

## ⚠️ 关键陷阱：帧数/fps ≠ 实际时长

**Wan I2V 的 VHS 输出实际时长 ≠ frames/fps 的数学结果。** 不要用 `80帧÷16fps=5.0s` 来计算 adelay。

实测验证（16fps 输出）：
| 设定帧数 | 理论时长 | 实际时长 (ffprobe) | 误差 |
|----------|---------|-------------------|------|
| 80 | 5.00s | 4.81s | -0.19s |
| 64 | 4.00s | 3.81s | -0.19s |
| 48 | 3.00s | 2.81s | -0.19s |

**正确流程**：
```bash
# 1. 全部视频下载完成后，逐镜 ffprobe 实测时长
for f in 06-video/ep01_shot*_anim_v1.mp4; do
  d=$(ffprobe -v quiet -show_entries format=duration -of csv=p=0 "$f")
  printf "%-35s %7.3fs\n" "$(basename "$f")" "$d"
done

# 2. 用实际时长（非帧数/fps）计算累计 adelay
#    例: shot05 = shot01 + shot02 + shot03 的实际时长之和 × 1000
#    4.81 + 3.81 + 4.81 = 13.43s → adelay=13430

# 3. 用实际值重跑 ffmpeg 拼接
```

**为什么会偏移**：VHS_VideoCombine 的编码器可能引入微小的帧率转换误差，即使设定为 16fps，实际输出也有偏差。所有 Wan I2V 工作流（smoothMix、GGUF、官方 fp8）都受影响。
